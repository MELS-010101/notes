# 📘 Мой базовый справочник по Git для разработчика

В этом репозитории собраны основные команды Git, которые необходимы для ежедневной работы над проектами и корректного взаимодействия с GitHub.

## 🛠 Жизненный цикл изменений (Workflow)


| Команда | Описание действия | Когда использовать? |
| :--- | :--- | :--- |
| **`git init`** | Инициализация репозитория | Когда создаешь новый локальный проект. |
| **`git clone`** | Клонирование проекта | Чтобы скачать существующий репозиторий с GitHub. |
| **`git status`** | Проверка состояния | Чтобы увидеть, какие файлы изменены. |
| **`git add .`** | Индексация (Staging) | Чтобы подготовить все измененные файлы к сохранению. |
| **`git commit -m`** | Фиксация изменений | Чтобы сохранить текущее состояние кода. |

## 🌿 Работа с ветками и синхронизация


| Команда | Описание действия | Контекст |
| :--- | :--- | :--- |
| **`git branch`** | Список веток | Посмотреть, на какой ветке ты сейчас находишься. |
| **`git checkout -b`** | Создание новой ветки | Для разработки фичи отдельно от ветки `main`. |
| **`git merge`** | Слияние веток | Чтобы объединить код задачи в основную ветку. |
| **`git push`** | Публикация в Remote | Отправить локальные коммиты на сервер GitHub. |
| **`git pull`** | Обновление локальной копии | Скачать свежие изменения коллег из облака. |

## 🆘 Исправление ошибок и полезные инструменты


| Команда | Что происходит? | Зачем это нужно? |
| :--- | :--- | :--- |
| **`git log --oneline`** | История коммитов | Посмотреть список последних сохранений в кратком виде. |
| **`git diff`** | Просмотр разницы | Увидеть конкретные строки, которые были изменены. |
| **`git revert`** | Отмена через новый коммит | Безопасно откатить изменения, сохранив историю. |
| **`git reset --hard`** | Жесткий сброс | Удалить несохраненные правки и вернуться к коммиту. |

---
> 💡 **Junior Tip:** Никогда не делай `git push --force` в основную ветку (`main`), если работаешь в команде. Это может стереть чужую работу!
