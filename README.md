# 📘 Мой базовый справочник по Git

В этом репозитории собраны основные команды Git, которые необходимы для ежедневной работы над проектами и корректного взаимодействия с GitHub.

## 🛠 Жизненный цикл изменений (Workflow)


| Команда | Описание действия | Когда использовать? |
| :--- | :--- | :--- |
| [**`git init`**](https://git-scm.com) | Инициализация репозитория | Когда создаешь новый локальный проект. |
| [**`git clone`**](https://git-scm.com) | Клонирование проекта | Чтобы скачать существующий репозиторий с GitHub. |
| [**`git status`**](https://git-scm.com) | Проверка состояния | Чтобы увидеть, какие файлы изменены или еще не отслеживаются. |
| [**`git add .`**](https://git-scm.com) | Индексация (Staging) | Чтобы подготовить все измененные файлы к сохранению. |
| [**`git commit -m`**](https://git-scm.com) | Фиксация изменений | Чтобы сохранить «снимок» (snapshot) текущего состояния кода. |

## 🌿 Работа с ветками и синхронизация


| Команда | Описание действия | Контекст |
| :--- | :--- | :--- |
| [**`git branch`**](https://git-scm.com) | Список веток | Посмотреть, на какой ветке ты сейчас находишься. |
| [**`git checkout -b`**](https://git-scm.com) | Создание новой ветки | Для разработки фичи отдельно от стабильной версии (`main`). |
| [**`git merge`**](https://git-scm.com) | Слияние веток | Чтобы объединить код из выполненной задачи в основную ветку. |
| [**`git push`**](https://git-scm.com) | Публикация в Remote | Отправить свои локальные коммиты на сервер GitHub. |
| [**`git pull`**](https://git-scm.com) | Обновление локальной копии | Скачать свежие изменения коллег из удаленного репозитория. |

## 🆘 Исправление ошибок и полезные инструменты


| Команда | Что происходит? | Зачем это нужно? |
| :--- | :--- | :--- |
| [**`git log --oneline`**](https://git-scm.com) | История коммитов | Посмотреть список последних сохранений в кратком виде. |
| [**`git diff`**](https://git-scm.com) | Просмотр разницы | Увидеть конкретные строки, которые были изменены. |
| [**`git revert`**](https://git-scm.com) | Отмена через новый коммит | Безопасно откатить изменения, сохранив историю. |
| [**`git reset --hard`**](https://git-scm.com) | Жесткий сброс | Полностью удалить несохраненные правки и вернуться к коммиту. |

---
> 💡 **Junior Tip:** Никогда не делай `git push --force` в основную ветку (`main`), если работаешь в команде. Это может стереть чужую работу!
